<p>&lt;h1 id=“label-The+Bike+programming+language”&gt;The Bike programming language&lt;span&gt;&lt;a href=“#label-The+Bike+programming+language”&gt;&amp;para;&lt;/a&gt; &lt;a href=“#documentation”&gt;&amp;uarr;&lt;/a&gt;&lt;/span&gt;&lt;/h1&gt;</p>

<p>&lt;p&gt;&lt;a href=“christopherdumas.github.io/Bike”&gt;Official Website&lt;/a&gt;&lt;/p&gt;</p>

<p>&lt;p&gt;&lt;a href=“<a href=“Contact”>christopherdumas@me.com“>Contactchristopherdumas@me.com”>Contact”>christopherdumas@me.com“>Contact> author&lt;/a&gt; or @christopherdumas&lt;/p&gt;</p>

<p>&lt;p&gt;The Bike programming language is a programming language that combines Haskell&amp;#39;s currying, function composition, anonymous functions, and array comprehension, with Scala&amp;#39;s packages, classes, and traits, Python&amp;#39;s is and isnt style operators, Scheme&amp;#39;s everything is an expression idea, Ruby&amp;#39;s everything-is-an-object philosophy, and Go&amp;#39;s no paren control structures, with Coffeescript&amp;#39;s infix if and unless. It also has optional no-paren function calls, and immutable arrays are also present. This is mixed in with mixins and Java&amp;#39;s ananymous Classes, as well as JavaScript&amp;#39;s Objects (called hashes) and ES6&amp;#39;s destructuring assignment. It is immutable in everything by default, but in a way that makes it fairly painless to use.&lt;/p&gt;</p>

<p>&lt;p&gt;Bike&amp;#39;s motto is:&lt;/p&gt;</p>

<p>&lt;blockquote&gt; &lt;p&gt;Not a lot of syntax, but a lot more sugar.&lt;/p&gt; &lt;/blockquote&gt;</p>

<p>&lt;h2 id=“label-Roadmap”&gt;Roadmap&lt;span&gt;&lt;a href=“#label-Roadmap”&gt;&amp;para;&lt;/a&gt; &lt;a href=“#documentation”&gt;&amp;uarr;&lt;/a&gt;&lt;/span&gt;&lt;/h2&gt;</p>

<p>&lt;p&gt;In the &lt;a href=“TODO_md.html”&gt;TODO.md&lt;/a&gt; file is our roadmap to 1.0. It will be updated every major version release. Our versioning system is like this: The whole number (the 1 in 1.0) is incremented each major, or breaking version. The decimal point is incremented by five, and normally for big, but not breaking changes, like adding new operators or shorthand syntax. The Beta version (indicated by +b&amp;lt;number&amp;gt;) is incremented every time a bug is fixed, or a change is being contemplated but I have not fully committed to it.&lt;/p&gt;</p>
