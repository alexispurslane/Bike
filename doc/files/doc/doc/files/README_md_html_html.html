        &lt;h1 id=&quot;label-The+Bike+programming+language&quot;&gt;The Bike programming language&lt;span&gt;&lt;a href=&quot;#label-The+Bike+programming+language&quot;&gt;&amp;para;&lt;/a&gt; &lt;a href=&quot;#documentation&quot;&gt;&amp;uarr;&lt;/a&gt;&lt;/span&gt;&lt;/h1&gt;

        &lt;p&gt;&lt;a href=&quot;christopherdumas.github.io/Bike&quot;&gt;Official Website&lt;/a&gt;&lt;/p&gt;

        &lt;p&gt;&lt;a href=&quot;mailto:christopherdumas@me.com&quot;&gt;Contact author&lt;/a&gt; or
        @christopherdumas&lt;/p&gt;

        &lt;p&gt;The Bike programming language is a programming language that combines
        Haskell&amp;#39;s currying, function composition, anonymous functions, and
        array comprehension, with Scala&amp;#39;s packages, classes, and traits,
        Python&amp;#39;s &lt;code&gt;is&lt;/code&gt; and &lt;code&gt;isnt&lt;/code&gt; style operators,
        Scheme&amp;#39;s everything is an expression idea, Ruby&amp;#39;s
        everything-is-an-object philosophy, and Go&amp;#39;s no paren control
        structures, with Coffeescript&amp;#39;s infix &lt;code&gt;if&lt;/code&gt; and
        &lt;code&gt;unless&lt;/code&gt;. It also has optional no-paren function calls, and
        immutable arrays are also present. This is mixed in with mixins and
        Java&amp;#39;s ananymous Classes, as well as JavaScript&amp;#39;s Objects (called
        hashes) and ES6&amp;#39;s destructuring assignment. It is immutable in
        everything by default, but in a way that makes it fairly painless to use.&lt;/p&gt;

        &lt;p&gt;Bike&amp;#39;s motto is:&lt;/p&gt;

        &lt;blockquote&gt;
        &lt;p&gt;Not a lot of syntax, but a lot more sugar.&lt;/p&gt;
        &lt;/blockquote&gt;

        &lt;p&gt;Some code samples are in order:&lt;/p&gt;

        &lt;pre&gt;class Animal {&amp;#x000A;  def sound = &amp;quot;some strange animal sound&amp;quot; # define function that returns a string.         &lt;h2 id=&quot;label-Roadmap&quot;&gt;Roadmap&lt;span&gt;&lt;a href=&quot;#label-Roadmap&quot;&gt;&amp;para;&lt;/a&gt; &lt;a href=&quot;#documentation&quot;&gt;&amp;uarr;&lt;/a&gt;&lt;/span&gt;&lt;/h2&gt;

        &lt;p&gt;In the &lt;a href=&quot;TODO_md.html&quot;&gt;TODO.md&lt;/a&gt; file is our roadmap to 1.0. It
        will be updated every major version release. Our versioning system is like
        this: The whole number (the 1 in 1.0) is incremented each major, or
        breaking version. The decimal point is incremented by five, and normally
        for big, but not breaking changes, like adding new operators or shorthand
        syntax. The Beta version (indicated by &lt;code&gt;+b&amp;lt;number&amp;gt;&lt;/code&gt;) is
        incremented every time a bug is fixed, or a change is being contemplated
